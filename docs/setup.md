# Налаштування Pisun Bot

Цей посібник допоможе вам правильно налаштувати бота в Telegram та запустити його.

## 1. Створення бота (@BotFather)

1. Відкрийте бота [@BotFather](https://t.me/BotFather) в Telegram.
2. Напишіть команду `/newbot`.
3. Введіть ім'я бота (наприклад: `Мій Гігантський Бот`).
4. Введіть юзернейм бота (має закінчуватися на `bot`, наприклад: `MiyPisunBot`).
5. Скопіюйте отриманий **HTTP API Token**. Це ваш `BOT_TOKEN`.

## 2. Налаштування команд (Автоматично)

Бот автоматично додає команди в меню при кожному запуску.
Однак, якщо ви хочете налаштувати їх вручну через @BotFather (наприклад, щоб вони відображалися навіть коли бот вимкнений):

1. Пишіть `/mybots` у @BotFather.
2. Виберіть вашого бота.
3. Натисніть **Edit Bot** -> **Edit Commands**.
4. Надішліть наступний список:
   ```
   pisun - Поміряти пісюн (раз на день)
   top - Топ гігантів чату
   start - Інформація про бота
   ```

## 3. Налаштування приватності (Важливо!)

За замовчуванням боти не бачать повідомлень у групах, якщо до них не звертаються прямо. Для цього бота це **нормально**, оскільки він працює через команди (`/`), але якщо ви захочете, щоб він реагував на текст, потрібно вимкнути приватність.

Наразі **змінювати нічого не потрібно**, бот працює через слеш-команди.

Але переконайтеся, що бот має право:
- **Send Messages** (Писати повідомлення)
- **Inline Queries** (опціонально, наразі не використовується)

### Додавання в групи
Бот має бути адміністратором? **Ні**, звичайних прав достатньо.
Головне, щоб у налаштуваннях групи йому було дозволено писати повідомлення.

## 4. Запуск (Технічна частина)

1. Перейменуйте файл `.env.example` на `.env`:
   ```bash
   cp .env.example .env
   ```
2. Відкрийте `.env` і вставте токен:
   ```bash
   BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
   ```
3. Запустіть через Docker:
   ```bash
   docker-compose up -d --build
   ```

## 5. Перевірка

Додайте бота в групу і напишіть `/pisun`.
Якщо бот відповів — все працює! 🍌
